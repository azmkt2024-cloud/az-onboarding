## 1. 배경 & 문제 정의

보험설계사로 처음 입문하는 **무경력 신입**은 다음 장벽에 부딪힙니다.

1. **자격시험의 막막함** — 손해·생명·변액보험 시험의 일정·범위·공부법 정보가 흩어져 있음
2. **실무 도구 진입장벽** — 보험사별 전산 포탈, 보장분석, 보상 처리 등 어디서 시작할지 모름
3. **체계적 교육 부재** — 선배의 단편적 구두 전달에 의존, 표준화된 온보딩 자료 없음

에즈금융서비스는 이를 **단일 허브**로 묶어, 신입이 "첫걸음부터 프로가 되도록" 돕는 학습·실무 공간을 제공합니다.

---

## 2. 목표 (Goals)

| 구분 | 목표 |
|------|------|
| 핵심 | 신입 설계사가 **자격시험 합격 → 실무 투입**까지 한 곳에서 준비 |
| 사용자 가치 | 흩어진 정보를 모으고, 시험 핵심을 압축 제공하여 학습 시간 단축 |
| 운영 가치 | 어드민이 콘텐츠(요점정리·전산링크·Q&A·교육자료)를 코드 수정 없이 관리 |
| 자동화 | 시험 일정을 협회 사이트에서 **자동 수집**하여 항상 최신 상태 유지 |

### 비목표 (Non-Goals)
- 일반 고객(소비자) 대상 보험 판매·가입 기능 — 본 제품은 **내부 신입 교육용**
- 실제 계약 체결·결제 기능
- 모바일 네이티브 앱 (반응형 웹으로 대응)

| 페르소나 | 설명 | 주요 니즈 |
|----------|------|-----------|
| **신입 설계사** (주 사용자) | 보험업 무경력, 자격시험 준비 중 | 시험 일정·핵심 요약·모의고사·실무 도구 접근 |

## 4. 기술 스택 & 아키텍처

| 영역 | 기술 |
|------|------|
| 프레임워크 | Next.js 15 (App Router) · React 19 · TypeScript |
| 스타일 | Tailwind CSS (HTML 프로토타입의 CSS 변수 토큰 이식) |
| 백엔드/DB | Supabase (Postgres + Auth + Storage) |
| 배포 | Vercel (main 브랜치 push → 자동 배포) |
| 크롤링 | cheerio (협회 사이트 파싱) + Vercel Cron |
| AI | Anthropic Claude API (오프라인 Q&A 재작성 배치, 향후 PDF 문제 파싱) |

### 공개/인증 구조
- **공개 페이지**: 인증 불필요 — 메인 허브 및 모든 학습 콘텐츠
- **어드민** (`/admin/*`): Supabase Auth 인증 필수, `middleware.ts`로 보호

### 콘텐츠 전략
- **정적 요약집**: `public/summaries/*.html` — 앱과 동일한 디자인 토큰을 쓰는 완성형 학습 가이드, iframe 전체화면으로 제공
- **동적 콘텐츠**: Supabase DB (시험일정·Q&A·교육자료·문제)

## 5. 기능 명세 (Features)

### 5.1 구현 완료 ✅

#### F1. 메인 허브 (`/`)
- 단일 페이지 + 오버레이/모달 방식 (Toss 스타일 풀블리드 배경 밴드)
- 구성: Hero → 자격시험 → 요점정리 → 보험사 전산 → 보상 Q&A → Footer
- 시험 일정은 서버에서 **DB 우선** 조회(손보: 어드민 입력 / 생보: 크롤러 수집), 없으면 fallback

#### F2. 자격시험 정보 (ExamOverlay)
- 설계사 자격시험 안내(오버레이), 기출문제·실전 모의고사 진입 카드
- 시험 일정 표시 (손해/생명)

#### F3. 과목별 요점정리 (SummaryOverlay)
- 손해보험(`fire.html`, 38p) · 생명보험(`life.html`, 38p) · **변액보험(`variable.html`, 30p)** 3종 완비
- 각 과목 카드 클릭 → 전체화면 iframe 요약집

#### F4. 보험사 전산 포탈 (PolicyOverlay / InsurerModal)
- 보험사별 전산 바로가기, 보장분석 동의 모달, 관련 기관 링크

#### F5. 보상 Q&A (QnaOverlay / FaqModal)
- 보상 관련 질문 카드 → 아코디언 답변
- 일반 버전 + **쉬운 설명 버전** 2종 데이터 (`compensationQna`, `compensationQnaEasy`)

#### F6. 시험 일정 자동 크롤러
- KLIA(생명보험협회) · KNIA(손해보험협회) 크롤러 (`lib/crawlers/`)
- Vercel Cron으로 매주 월 06:00 자동 수집 (`/api/cron/exam-schedule`, `CRON_SECRET` 인증)

#### F7. 어드민 — 시험 일정 관리 (`/admin/exam-schedule`)
- 로그인(`/admin/login`) + 보호 레이아웃
- 손해보험 시험 일정 수동 입력/편집 (LossScheduleManager)

### 5.2 진행 중 / 부분 구현 🔧
- 어드민 CMS 확장: 요점정리·전산링크·보상 Q&A 편집 화면 (`SKILL-admin.md`)
- Supabase 환경변수 연동 (현재 `.env.local` 미설정 → 정적·fallback 콘텐츠로 동작)

### 5.3 계획 (Roadmap) 📋

#### F8. 기출문제 PDF 업로드·뷰어 (`SKILL-pdf.md`)
- 어드민이 기출 PDF 업로드 → Supabase Storage → 공개 뷰어 제공

#### F9. 실전 모의고사 엔진 (`SKILL-quiz.md`)
- CBT 방식 (`/mock`): 손해 50문항/50분 · 생명 40/40 · 변액 45/45
- 타이머, 문제 번호판, 즉시 채점, 과목별 정답률, 오답 노트
- `quiz_questions` / `mock_sessions` 테이블, 정답은 **서버사이드 검증만**
- PDF → Claude API 자동 문제 파싱 → 어드민 검토 후 승인

#### F10. 보험 기초 교육 메뉴 (`SKILL-education.md`)
- `/education`: 카테고리별 PDF/영상/아티클 콘텐츠 허브
- `edu_categories` / `edu_materials` 테이블 + `edu-materials` Storage 버킷
- 어드민 업로드(드래그 정렬), 아티클 마크다운 렌더링

## 8. 비기능 요구사항

- **언어**: 한국어 UI / TypeScript 코드
- **반응형**: 모바일·데스크톱 대응 (신입은 주로 모바일 학습)
- **성능**: 요약집 iframe은 오버레이가 **열렸을 때만 로드** (동시 로딩 방지)
- **보안**:
  - `SUPABASE_SERVICE_ROLE_KEY`·`ANTHROPIC_API_KEY` 등 서버 전용 키 클라이언트 노출 금지
  - 모의고사 정답/해설 클라이언트 상태에 포함 금지, 서버 검증
  - `.env.local` 절대 커밋 금지
- **운영**: 어드민이 비개발자도 콘텐츠 관리 가능해야 함
